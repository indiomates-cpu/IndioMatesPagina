Industria Matera — Sitio de E-commerce

Documento de referencia con el alcance, stack, modelo de datos y hoja de ruta acordados para el desarrollo del sitio. Pensado para cargarse en la carpeta de trabajo del proyecto como punto de partida.

1. Resumen funcional

Catálogo público

  • No requiere cuenta de usuario para navegar.
  • Home con productos destacados y acceso a las 10 categorías definidas.
  • Listado de productos (~50 en total) filtrable por categoría, con imagen de portada, nombre, precio y stock disponible visible en cada tarjeta.
  • Vista de detalle por producto: descripción, precio, galería con múltiples imágenes (sin límite fijo, a criterio del administrador), cantidad exacta de stock y selector de cantidad limitado a lo disponible.
  • Indicadores automáticos de stock: aviso de "sin stock" en 0 unidades, y aviso destacado cuando queda poco (umbral a definir).

Carrito y pedido

  • Agregar, quitar y modificar cantidades de productos en el carrito.
  • No procesa pagos online.
  • Al finalizar, arma un resumen del pedido y lo envía como mensaje prellenado a WhatsApp — la venta se cierra por ese canal.

Panel de administración

  • Acceso con login (no público).
  • Alta, edición y baja lógica de productos.
  • Carga y gestión de múltiples imágenes por producto, con reordenamiento.
  • Edición manual del stock, reflejado en tiempo real en el catálogo público.
  • Gestión de categorías.
  • Panel de métricas propio (productos más vistos, pedidos enviados por WhatsApp, productos sin stock), complementado con Google Analytics para tráfico general.

Identidad visual

  • Paleta principal: blanco y negro.
  • Animaciones con temática de mate y cultura argentina en distintos puntos de la experiencia.
  • Logo pendiente de incorporar — no bloquea el desarrollo del resto del sitio.

3. Modelo de entidades (Prisma)

// ============================
// CATEGORÍA
// ============================
model Categoria {
  id        String     @id @default(cuid())
  nombre    String
  slug      String     @unique
  orden     Int        @default(0)   // orden de visualización en el menú
  productos Producto[]
  creadoEn  DateTime   @default(now())
}

// ============================
// PRODUCTO
// ============================
model Producto {
  id            String            @id @default(cuid())
  nombre        String
  slug          String            @unique
  descripcion   String
  precio        Decimal
  stock         Int               @default(0)
  activo        Boolean           @default(true)   // visible/oculto en el catálogo
  categoriaId   String
  categoria     Categoria         @relation(fields: [categoriaId], references: [id])
  imagenes      ImagenProducto[]
  eventos       EventoMetrica[]
  creadoEn      DateTime          @default(now())
  actualizadoEn DateTime          @updatedAt
}

// ============================
// IMAGEN DE PRODUCTO (relación 1:N)
// ============================
model ImagenProducto {
  id          String   @id @default(cuid())
  url         String
  orden       Int      @default(0)   // 0 = portada
  productoId  String
  producto    Producto @relation(fields: [productoId], references: [id], onDelete: Cascade)
}

// ============================
// ADMINISTRADOR
// ============================
model Admin {
  id             String   @id @default(cuid())
  usuario        String   @unique
  contrasenaHash String
  creadoEn       DateTime @default(now())
}

// ============================
// EVENTO DE MÉTRICA (panel propio)
// ============================
model EventoMetrica {
  id          String    @id @default(cuid())
  tipo        String    // ej: "vista_producto", "pedido_whatsapp"
  productoId  String?
  producto    Producto? @relation(fields: [productoId], references: [id])
  fecha       DateTime  @default(now())
}
Notas de diseño: el campo slug en Producto y Categoria genera URLs amigables y ayuda al SEO. El campo activo permite ocultar un producto sin borrar su historial. onDelete: Cascade en ImagenProducto evita huérfanos en la base de datos al eliminar un producto (los archivos en Cloudinary requieren borrado explícito aparte). EventoMetrica usa tipo como string libre para poder sumar nuevos eventos sin modificar el esquema.

4. Paso a paso

Fase 1 — Modelado y diseño (2-3 días)

  • Definir el modelo de datos (ver sección 3).
  • Definir identidad visual: paleta blanco/negro, tipografía, y qué elementos de cultura mate se van a animar.
  • Wireframes de Home, catálogo por categoría, detalle de producto, carrito y panel admin.

Fase 2 — Base del proyecto (1 día)

  • Crear proyecto Next.js con TypeScript y Tailwind.
  • Configurar Prisma + PostgreSQL (Supabase/Railway) y correr la primera migración.
  • Configurar Cloudinary.
  • Definir variables de entorno.

Fase 3 — Catálogo público (3-4 días)

  • Home con productos destacados y animación inicial.
  • Listado de productos por categoría, con stock visible en cada tarjeta.
  • Vista de detalle con galería de imágenes (miniaturas o carrusel, lazy loading).
  • Micro-animaciones: hover en tarjetas, transición de galería, ícono de mate en el header.

Fase 4 — Carrito y pedido por WhatsApp (2 días)

  • Estado global del carrito (Context API o Zustand).
  • Vista de carrito con resumen de productos, cantidades y subtotal.
  • Botón "Enviar pedido por WhatsApp" que arma el mensaje y redirige a wa.me.
  • Pantalla de confirmación tras el envío.

Fase 5 — Panel de administración (4-5 días)

  • Login para administradores.
  • CRUD de productos, incluyendo carga y reordenamiento de múltiples imágenes.
  • Gestión de stock manual, reflejado en tiempo real en el catálogo.
  • Gestión de categorías.
  • Panel de métricas propio.

Fase 6 — Analítica externa (medio día)

  • Integrar Google Analytics 4.

Fase 7 — Pulido visual y animaciones (2-3 días)

  • Consistencia del blanco y negro en todos los componentes.
  • Detalles de identidad argentina/mate (loader, transiciones, ícono de carrito vacío).
  • Responsive completo, mobile-first.

Fase 8 — Pruebas (1-2 días)

  • Flujo completo: navegación → carrito → envío del pedido por WhatsApp.
  • Panel admin: stock reflejado en tiempo real.
  • Pruebas en distintos dispositivos y navegadores.

Fase 9 — Despliegue (1 día)

  • Deploy en Vercel.
  • Base de datos en producción.
  • Configurar dominio (propio o subdominio gratuito de Vercel).
  • Verificar Analytics y link de WhatsApp con datos reales.

Estimación total: entre 3 y 4 semanas de desarrollo part-time.

5. Lógica de visualización de stock

El campo stock (entero) ya almacena la cantidad exacta. La lógica de presentación en el frontend:

if (stock === 0) → "Sin stock" (deshabilita el botón de agregar al carrito)
else if (stock <= UMBRAL_BAJO) → "¡Últimas {stock} unidades!" (destacado)
else → "Stock: {stock} unidades"

El valor de UMBRAL_BAJO queda pendiente de definir (sugerido: 5 unidades). En la vista de detalle, el selector de cantidad no debe permitir superar el stock disponible.

6. Pedido por WhatsApp

El carrito no procesa pagos. Al confirmar el pedido, se genera un link de tipo:

https://wa.me/NUMERO_DEL_NEGOCIO?text=MENSAJE_CODIFICADO

El MENSAJE_CODIFICADO se arma dinámicamente con el detalle del pedido (productos, cantidades, subtotal) usando encodeURIComponent() sobre el texto generado. No requiere WhatsApp Business API ni ningún costo asociado.

7. Identidad visual

Paleta: blanco y negro como colores principales.
Temática: animaciones relacionadas al mate y la cultura argentina (vapor, yerba, bombilla, mate cebado) distribuidas en puntos clave de la experiencia.

El logo se incorporará más adelante; su ausencia no bloquea el desarrollo del resto del sitio.
