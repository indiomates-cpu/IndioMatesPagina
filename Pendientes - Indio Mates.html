Imprescindible para funcionar de punta a punta

1

Base de datos PostgreSQL

Obligatorio

Es el único bloqueante real. Sin una base conectada no se muestran productos ni categorías, no funciona el login del admin y no se registran métricas. El modelo de datos (Prisma) ya está definido; solo falta apuntar a una base y crear las tablas.

Opción A — Base gratis en la nube (recomendada, sin instalar nada)
  • Creá una cuenta en Neon (neon.tech) o Supabase (supabase.com) — ambos tienen plan gratuito.
  • Creá un proyecto y copiá la connection string (empieza con postgresql://).
  • Pegala en el archivo .env como valor de DATABASE_URL.
Opción B — PostgreSQL local
  • Instalá PostgreSQL en Windows (instalador oficial) o usá Docker si lo tenés.
  • Ajustá DATABASE_URL con tu usuario, contraseña y nombre de base.
Después, en ambos casos
# crea las tablas a partir del schema de Prisma
npm run db:push

# carga admin + 14 productos + categorías de prueba
npm run db:seed
Resultado
  • Catálogo con productos reales en localhost:3000.
  • Login del admin operativo (admin / indiomates2024).
  • Métricas registrando vistas y pedidos.
2

Número de WhatsApp real

Obligatorio

El checkout arma un link wa.me/NUMERO con el pedido. Hoy el número es un placeholder (5493764000000), así que los pedidos no llegarían al negocio.

Qué hacer
  • En .env, poné el número real en NEXT_PUBLIC_WHATSAPP_NUMBER.
  • Formato internacional, sin +, espacios ni guiones.
  • Ejemplo Argentina (celular): código país 54 + 9 + área sin el 0 + número sin el 15. → 5493764123456
Nota
  • Al ser una variable NEXT_PUBLIC_, después de cambiarla hay que reiniciar npm run dev.

Opcional / recomendado

3

Cloudinary (subida de imágenes por archivo)

Opcional

Habilita subir imágenes de producto desde archivo con optimización automática. Sin configurarlo, el panel igual funciona: se cargan imágenes por URL (el seed usa placeholders por URL).

Qué hacer
  • Creá una cuenta gratis en Cloudinary y completá en .env: CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET y NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME.
  • Al detectar credenciales, el formulario de producto muestra el botón de subir archivo automáticamente.
4

JWT_SECRET propio

Para producción

Firma los tokens de sesión del admin. El valor actual es de desarrollo: sirve en local, pero conviene uno propio y secreto antes de publicar.

# genera una clave aleatoria y pegala en JWT_SECRET

5

Credenciales del admin

Recomendado

Por defecto el seed crea admin / indiomates2024. Cambiá ADMIN_USUARIO y ADMIN_CONTRASENA en .env antes de correr el seed para usar tus propias credenciales.

Ruta mínima para verlo funcionando

  1. Conseguí una DATABASE_URL de PostgreSQL Neon o Supabase gratis (sin instalar), o PostgreSQL local.
  2. Pegala en el archivo .env Reemplazá el valor de DATABASE_URL.
